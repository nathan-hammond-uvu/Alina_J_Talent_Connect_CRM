Talent Connect CRM API

Build integrations with our JSON-based API. Read CRM records securely.

Introduction

The Talent Connect CRM API provides read access to all CRM resources using a clean, RESTful JSON interface.

Base URL:

https://your-host.example.com/api/v1

All responses are JSON. Each endpoint returns data scoped to the authenticated user's permissions.

Authentication

The API uses session-based authentication, the same as the web application.

Getting Started

  • Log in with your credentials
  • Your session cookie authenticates subsequent API requests
  • Requests without valid sessions return 401 Unauthorized

Security Notes

  • User passwords are never exposed in API responses
  • Out-of-scope or missing resources return 404 Not Found (no data leakage)
  • All data is filtered by role-based access control (RBAC)

Available Endpoints

Models

Each model has a list endpoint and a detail endpoint:

List
GET /api/v1/{model}
Detail
GET /api/v1/{model}/{id}

Supported Models

Use these model names in the endpoint URLs:

  • roles – System roles (Admin, Manager, etc.)
  • persons – Contact information (shared by users, employees, etc.)
  • users – User accounts (passwords omitted)
  • employees – Employee records
  • creators – Creator / Talent records
  • social_media_accounts – Creator social links
  • brands – Brand records
  • brand_contacts – Brand contact representatives
  • deals – Deal records (creator + brand)
  • contracts – Contract records

Legacy Endpoints

For backward compatibility:

GET /api/v1/items
GET /api/v1/items/{id}

These are aliases for /api/v1/creators

API Examples

List Creators

GET /api/v1/creators

Get a Single Creator

GET /api/v1/creators/11

List Deals

GET /api/v1/deals

Get a Specific Contract

GET /api/v1/contracts/5

Using cURL

curl -i https://your-host.example.com/api/v1/creators

Response Format

Success: List Response (200 OK)

{ "creators": [ { "id": 11, "creator_id": 11, "person_id": 10, "employee_id": 9, "name": "Nate", "description": "Test" } ] }

Success: Detail Response (200 OK)

{ "creator": { "id": 11, "creator_id": 11, "person_id": 10, "employee_id": 9, "name": "Nate", "description": "Test" } }

Error: Unauthorized (401)

{ "error": "Unauthorized" }

Error: Not Found (404)

{ "error": "Not found" }

Quick Start

Step 1: Log In

Visit the login page and authenticate with your credentials.

Step 2: Open Your API Client

Use curl, Postman, JavaScript fetch(), or any HTTP tool.

Step 3: Make Your First Request

curl -i https://your-host.example.com/api/v1/creators

Step 4: Parse the Response

Extract and use the JSON data in your application. All records include an id field for referencing and detail calls.

Common Patterns

  • Fetch all records: GET /api/v1/creators
  • Fetch one record: GET /api/v1/creators/11
  • Handle 404: Record is out of scope or doesn't exist
  • Handle 401: Session expired; log in again
